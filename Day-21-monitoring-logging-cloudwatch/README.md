## Monitoring vs Logging
- Monitoring tracks system performance
- Logging captures detailed event records

## Amazon CloudWatch
- Native AWS monitoring service
- Collects metrics and logs
- Triggers alarms based on thresholds

## Metrics
- Quantitative data points
- Examples: CPUUtilization, DiskReadOps
- Collected at regular intervals

## Logs
- Application and system logs
- Stored in log groups and streams
- Useful for debugging issues

## Alarms
- Trigger notifications
- Based on metric thresholds
- Can initiate auto scaling actions

## Basic Troubleshooting Approach
1. Check CloudWatch metrics
2. Review logs for errors
3. Identify bottlenecks
4. Apply corrective actions

## Key Takeaways
- CloudWatch provides visibility into AWS resources
- Metrics and logs help identify issues quickly
