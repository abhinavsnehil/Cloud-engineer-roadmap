## Backup
- Data copies stored for recovery
- Protects against data loss and corruption
- Regular and automated backups recommended

## Disaster Recovery
- Strategy to restore systems after failure
- Protects against region or AZ failures

## RTO (Recovery Time Objective)
- Maximum acceptable downtime
- Defines how quickly systems must be restored

## RPO (Recovery Point Objective)
- Maximum acceptable data loss
- Defines how much data loss is tolerable

## AWS Backup Options
- EBS snapshots
- RDS automated backups
- S3 versioning and replication

## DR Strategies
- Backup and Restore
- Pilot Light
- Warm Standby
- Multi-site active/active
