## IAM Roles
- Provide temporary credentials
- Used by AWS services and users
- More secure than access keys

## IAM Policies
- JSON-based permission documents
- Define allowed and denied actions

## Policy Types
- Managed policies
- Inline policies
- Resource-based policies

## AWS STS (Security Token Service)
- Issues temporary credentials
- Used for role assumption
- Enables cross-account access

## Cross-Account Access
- Secure access between AWS accounts
- Uses IAM roles and trust policies

## Best Practices
- Use least privilege
- Avoid long-term credentials
- Use roles wherever possible

- Reviewed IAM role trust relationships
- Examined policy structure
- Understood STS use cases

## Key Takeaways
- Roles and STS improve security
- Temporary credentials reduce risk
