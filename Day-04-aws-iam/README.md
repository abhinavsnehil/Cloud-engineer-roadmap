## What I Learned
- IAM controls access to AWS resources securely
- Users represent individuals, groups represent collections of users
- Roles are used by AWS services to access resources
- Policies define permissions using JSON

## IAM Components
- Users
- Groups
- Roles
- Policies

## Best Practices
- Never use root user for daily tasks
- Enable MFA on root and IAM users
- Grant least privilege permissions
- Rotate access keys regularly

## Practical Tasks
- Created IAM users and groups
- Attached policies to users
- Explored predefined AWS managed policies

## Key Takeaways
- IAM is the foundation of AWS security
- Proper IAM configuration prevents unauthorized access
